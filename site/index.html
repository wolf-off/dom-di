DOM-DI Project

Site: http://dom-di.org

NPM: dom-di

GitHub: https://github.com/wolf-off/dom-di

Purpose

Target of DOM-DI is support your frontend by low coupling.

It allow you to implement Interaction and Dependency Injection between your components.

There are two main dirrection of usage:

  • In Web-components
  • When you works with differents frameworks in one application (Angulars/React/Vue)

Anyway It allows you works in Micro Frontend approach

Install

All you need is in src folder Usage Initialize conversation

Initialisation

There are two way to start conversation between your components:

  • Add dom-di-container (/src/Controls/Container.html) to root of your document (or arround).
  • Or inherit one control from diContainerMixin (/src/diContainerMixin.js)

Sure that all partisipants of conversation is childe(may not dirrect) of the control

  `class MyControl extends diContainerMixin(HTMLElement) `

  `class MyControl extends diObjectMixinBabel(React.Component)`

  `class MyControl extends diObjectMixinBabel({})`

Provide Control

  • Inherit your control from diContainerMixin (/src/diContainerMixin.js)
  • Add

this.typeName = 'control-to-provide';

this.dependencies = [];

this.diReady = () => { return this; }

Receive Control

  • Inherit your control from diContainerMixin (/src/diContainerMixin.js)
  • Add

this.dependencies = ['control-to-provide'];

this.diReady = (control) => {

    // add your code

}

If you have done well, diReady will be called with control, with you want to provide
